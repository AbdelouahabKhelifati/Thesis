mco_trans_set_gc_policy

Set the garbage collection policy for the current connection.

Prototype

 
    MCO_GC_POLICY mco_trans_set_gc_policy( /*IN*/ mco_db_h db,
                         /*IN*/ MCO_GC_POLICY new_policy );
 

Arguments

db The database handle that was established by mco_db_connect()

new_policy

The policy to be set; it must be one of the following:

  • MCO_GC_DISABLED: garbage collection is disabled
  • MCO_GC_SELF_VERSIONS: each session will perform a cleanup of its own garbage only. This policy can be efficient when each session (connection) works with its own subset of classes (tables)
  • MCO_GC_ON_MODIFICATION: read-write transactions in any session will trigger garbage collection in all sessions (connections)
  • MCO_GC_ALWAYS: like MCO_GC_ON_MODIFICATION, but READ_ONLY transactions will trigger garbage collection as well

Description

This function sets the garbage collection policy for the given session (connection). It should be called only once when a database session (connection) is created, and before any transactions are started. Furthermore, all sessions (connections) should explicitly set the same policy, unless the default is used (an exception is made for MCO_GC_DISABLED).

The MCO_GC_DISABLED policy is intended to be used to limit garbage collection to dedicated sessions (connections). In this scenario, the worker sessions will have this policy, whereas a dedicated session (or multiple sessions) with policy MCO_GC_ALWAYS will collect garbage in separate threads.

The default policy is MCO_GC_ON_MODIFICATION.

Return Codes

MCO_GC_POLICY The previous garbage collection policy
